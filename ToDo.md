в VSCodium, для Svelte-проекта, чтобы не сканировать лишний мусор (картинки, кеш и модули):

Bash
grep -rEoh "#[0-9a-fA-F]{3,6}|rgba?\(.*?\)|hsla?\(.*?\)" --exclude-dir={node_modules,.svelte-kit,.git} . | sort | uniq

Что она сделает:
-r: Ищет рекурсивно во всех папках.
-Eoh: Использует расширенные регулярки, выводит только само совпадение (без лишнего текста) и скрывает имена файлов.
#...|rgba...: Ищет HEX, RGB и HSL цвета.
--exclude-dir: Игнорирует папку библиотек и служебную папку SvelteKit (чтобы не собрать тысячи цветов из чужих пакетов).
| sort | uniq: Сортирует их по алфавиту и удаляет дубликаты.

	   ❎   Продолжаем наполнять функционал калькулятора.
	 Тригонометрические функции.
	 Пример для вычисления синуса. Пользователь вводит число (например, 10) и нажимает клавишу нужной функции (например, sin) на экране (и в appState.display) попадёт строка с учётом значения в appState.corner (например, deg) - на экране будет строка sin(10deg).  Пользователь нажимает кнопку = и в appState.expression попадает строка sin(10deg) = 0.1736481777,  а в appState.display = "0". 2. Вопросы по интерфейсу и логике ввода (UX)
Поскольку мы отказываемся от пробела и обычных скобок для целой части, давайте утвердим правила ввода:

Пользователь будет вводить смешанную дробь таким образом:

Вариант А: Одной контекстной кнопкой ÷. Например: нажал 5, затем ÷ (появился маркер целой части 5⥑), затем 1, затем снова ÷ (получилось 5⥑1÷), затем 2 (итог: 5⥑1÷2⥏).

Для внутренней строки оставляем текущие спецсимволы ⥑ и ⥏ для отделения целой части внутри строки currentExpr. (Это очень удобно, так как они гарантированно не пересекаются с математическими скобками). Эти символы не видимы для пользователя.

2. Документация модуля fractionActions.js
Назначение
Сервис управляет логикой дробного калькулятора, преобразуя пользовательский ввод в математические операции.

Описание функций:
convertRawInputToMarkers: Служебная функция-транслятор. Находит в строке шаблоны смешанных дробей и помечает их специальными символами для парсера. Не меняет состояние дисплея.

fractionToString: Преобразует объект Fraction в строку для отображения, автоматически выделяя целую часть (напр., 11/2 -> 5 1/2).

tokenizeSubExpression: Превращает строковое выражение в массив объектов Fraction, необходимых для математических вычислений.

enterFractionDigit: Просто добавляет цифру в appState.display. Не вызывает парсинг, поэтому не вызывает ошибок при наборе.

handleFractionSlash: Добавляет символ деления (дробную черту) в текущее число на дисплее.

executeIdentity: Добавляет символ ( на дисплей для начала формирования смешанной дроби.

enterFractionOperator: Фиксирует текущее число с дисплея в appState.expression и очищает дисплей. Теперь не вызывает парсинг, предотвращая ERROR при наборе.

backspaceFraction: Удаляет последний введенный символ на дисплее.

calculateFractionResult: Главная функция. Собирает всё выражение, раскрывает смешанные дроби в неправильные, считает результат по правилам приоритета операций и обновляет дисплей. Только здесь выполняется математическая проверка.

clearFractionAll: Сбрасывает все состояния калькулятора в начальное.



Шаги для пояснения вычислений. Каждый из шагов может отсутствовать..
1. Условие - вводим дробь.
2. Считаем то что в скобках числителя и/или знаменателя, если есть.
3. Целые части переносим в числитель, если они есть.
4. Приводим все дроби к общему знаменателю, если дробей несколько, если оператор + или - .
5. Если оператор + или - то знаменатель общий, а вчислителе показываем строку, которую будем вычислять.
Если * то перемножить числители и перемножить знаменатели.
Если деление, то перевернуть дробь по мат правилам 
6. Полученная дробь. Проверяем возможность сокращения дроби и делаем если это возможно 
7. Выделяем целую часть из дроби, если это возможно.
8. Результат.
